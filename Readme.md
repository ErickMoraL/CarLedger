# CarLedger

CarLedger is a backend system for managing work vehicles such as taxis, Uber, or similar services.

Its goal is to help small vehicle owners or operators track:

- vehicle income
- operational expenses
- maintenance records
- movement history
- real profitability per vehicle

## Problem It Solves

In many cases, vehicle owners do not have a clear way to understand:

- how much each vehicle actually earns
- how much is spent on fuel, maintenance, and repairs
- when the next service is due
- whether a vehicle is still profitable or not

CarLedger addresses this by providing a structured and traceable way to manage this information.

## Project Goal

Build a simple, functional MVP that allows users to register and track the basic operation of one or multiple vehicles.

This project is also part of my professional portfolio, focused on backend development, data modeling, business logic, and Laravel.

## MVP Scope

The initial version includes:

- vehicle registration
- income tracking per vehicle
- expense tracking per vehicle
- maintenance records (date, mileage, cost)
- vehicle history overview
- total income, total expenses, and net profit calculation per vehicle
- basic view of upcoming maintenance

## Tech Stack

- **Backend:** Laravel 12
- **Database:** MySQL
- **Frontend:** Blade (simple UI) or basic API
- **Deployment:** Render

## Project Approach

CarLedger is being developed with a practical mindset:

- avoid overengineering
- focus on delivering a working MVP
- keep the architecture simple and clear
- build a real project suitable for interviews and GitHub

## Core Domain

The system revolves around these main entities:

- vehicles
- income
- expenses
- maintenance

From these, key insights are generated such as:

- vehicle history
- net profit over time
- accumulated expenses
- upcoming maintenance
